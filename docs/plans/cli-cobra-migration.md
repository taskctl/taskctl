# Stream A — Cobra migration + everything that falls out of rewriting the commands

## Context

Several CLI defects share one root cause: taskctl uses urfave/cli/v2, whose stdlib-`flag` parser stops at the first positional argument and has no persistent-flag inheritance. So `taskctl list --output json` fails (`flag provided but not defined: -output`), `taskctl build --raw` dies with `unknown task or pipeline "--raw"` (the flag is swallowed as a target name at `cmd/run.go:142`), and `<task> --summary=false` is silently ignored. Rather than patching around this with arg-hoisting, we migrate to **spf13/cobra (latest, v1.10.x)**: pflag intersperses flags/args natively and persistent flags are inherited by every subcommand — the whole bug class dissolves structurally, and cobra brings native completions, did-you-mean, and grouped help for free.

At the same time the `cockpit` output becomes the flagship: renamed to **`default`** and made the default format on a TTY. Because the dashboard shows only a spinner + "Finished" lines (never command stdout), the end-of-run **summary becomes the primary artifact** users are left with — so it gets substantially enriched and extended to single-task runs (see Stream B).

Facts established during exploration (verified against source):

- Global flags: `cmd/cmd.go:80-133` (`--debug/-d`, `--config/-c`, `--output/-o`, `--raw/-r`, `--cockpit`, `--quiet/-q`, `--set`, `--dry-run`, `--summary/-s` default `true`, `--no-input`). Resolution in the app `Before` hook `cmd/cmd.go:134-179`; default is `prefixed` (`cmd.go:165-166`); cockpit→prefixed downgrade on non-TTY (`cmd.go:174-176`).
- `run` re-declares `--dry-run`/`--summary` locally (`cmd/run.go:31-42`) — duplication to remove.
- Format constants: `internal/output/output.go:11-16`; decorator dispatch `output.go:45-64`. Cockpit dashboard: `internal/output/cockpit.go` — inline bubbletea (no alt-screen), process-wide singleton, torn down via `output.Close()` ← `runner.TaskRunner.Finish()` (`runner/runner.go:207`).
- Summary today: `printSummary` `cmd/run.go:316-346`, called from `runPipeline` `run.go:152-174` **after** the dashboard exits (guarded `cfg.Output != FormatJSON`); pipelines only — `runTask` (`run.go:289-298`) never prints one.
- Completion today: hand-rolled `cmd/completion.go` printing urfave wrapper scripts; artifacts in `autocomplete/`; regenerated by the `update-completers` task (`tasks.yaml:63-69`); dynamic names via app-level `BashComplete` (`cmd.go:69-79`).
- urfave APIs in use (migration inventory): `cli.Context` everywhere, `c.Args()/.Bool()/.String()/.IsSet()/.NArg()/.StringSlice()`, app `Before`, per-command `Before`/`After` (run/watch/validate), `cli.ShowCommandHelp` (graph.go:30), `BashComplete`, `cli.Author`. **Not** used: `ExitCoder`, `altsrc`, custom flag types.
- `--` handling: `targetNames`/`taskArgs` (`run.go:179-192`, `300-314`) rely on the literal `--` surviving in `c.Args()`; cobra strips it — use `cmd.ArgsLenAtDash()` instead.

User decisions: one combined effort split into sequential streams; summary = counts header + per-task status/duration + times + exit codes + output size + failed-task log tails + everything else we know, extended to single tasks; non-TTY fallback = `prefixed`; extras = native completions, did-you-mean, grouped styled help, live output in the dashboard.

## Stream A — Cobra migration + everything that falls out of rewriting the commands

*(One stream: the package reorganization, flag fixes, the `default` rename, `--cockpit` removal, completions, did-you-mean, and grouped help all live inside the code being rewritten anyway.)*

### A0. Package layout — one directory per command

Split the flat `cmd` package into per-command packages (user-selected layout):

```
cmd/
  root.go            package cmd — root command, persistent flags, PersistentPreRunE, rootRunE, listenSignals
  run/run.go         package run — run command + `run task` subcommand + shared run orchestration (see below)
  list/list.go       package list — list + tasks/pipelines/watchers subcommands
  show/show.go       package show
  watch/watch.go     package watch
  graph/graph.go     package graph
  init/init.go       package initcmd  (dir `init/`, package named initcmd — `package init` is not importable)
  validate/validate.go  package validate
  skill/skill.go     package skill
internal/cmdutil/    shared plumbing: config load + ErrConfigNotFound tolerance, --set application,
                     output-format resolution, env binding, TaskRunner construction, buildSuggestions,
                     nonInteractive, stdin seam (Stdin/SetStdin), splitArgsAtDash
```

- Shared state moves off package globals into `cmdutil.State` (fields: `Cfg *config.Config`, `Loader *config.ConfigLoader`, `Stdin io.ReadCloser`, `Version string`, `DryRun/Quiet/...` resolved flags as needed). Each command constructor takes it: `run.NewCommand(st *cmdutil.State) *cobra.Command`, `list.NewCommand(st)`, etc. Root builds one `State` and hands it to every constructor — no cross-package mutable globals, and tests get isolated state.
- Run orchestration is needed by both the root (bare `taskctl <task>`, interactive selector) and the `run` command, so `cmd/run` exports it: `run.RunTargets(st, cmd, targets []string) error`, `run.RunTask(...)`, `run.RunPipeline(...)`, plus the NDJSON event emitters (`emitRunStarted`/`emitRunFinished`) and (Stream B) the summary. `cmd` imports `cmd/run` — no cycle, `run` never imports `cmd`.
- `cmd.SetSkillTemplate` (called from `main.go` go:embed injection) moves to `skill.SetTemplate` or stays as a `cmd`-level forwarder — keep `main.go`'s call site working with a one-line change at most.
- Existing tests move with their commands (`cmd/list/list_test.go`, …); shared test helpers (app runner, `SetStdin` seam) land in `cmdutil` or a `cmdutil/cmdtest` helper.

### A1. Dependency + root command skeleton

- `go get github.com/spf13/cobra@latest` (pflag comes transitively). Remove `github.com/urfave/cli/v2` when the last usage is gone; `go mod tidy`.
- `cmd/root.go` (replaces `cmd/cmd.go`):
  - `func Run(version string) error` — keeps signature (called from `main.go:24`). Creates a cancelable context, starts `listenSignals` (now cancels the context instead of the `cancelFn` global — drop `cancelFn`/`cancelMu`), builds the root via `NewRootCommand(version)`, runs `root.ExecuteContext(ctx)`.
  - `func NewRootCommand(version string) *cobra.Command` — replaces `NewApp`. Root: `Use: "taskctl [target...] [-- task-args]"`, `Short: "modern task runner"`, `Version: version`, `Args: cobra.ArbitraryArgs`, `SilenceUsage: true`, `SilenceErrors: true` (main.go keeps `slog.Error` + `os.Exit(1)`), `RunE: rootRunE`. Builds the `cmdutil.State` and registers all subcommand constructors.
- **Persistent flags** (on root, inherited everywhere — this alone fixes #4/#5/#6/#7):
  - `--debug/-d` bool, `--config/-c` string (default text "tasks.yaml or taskctl.yaml"), `--output/-o` string (usage: `output format (default, prefixed, raw or json)`), `--raw/-r` bool ("shortcut for --output=raw"), `--quiet/-q` bool, `--set` string slice, `--dry-run` bool, `--summary/-s` bool default `true`, `--no-input` bool.
  - **`--cockpit` is removed** (item 3). No alias, no deprecation stub — it errors as unknown flag; the release notes/README explain the rename.
  - Env binding helper (pflag has no EnvVars): `func bindFlagEnv(fs *pflag.FlagSet, name string, envVars ...string)` — in `PersistentPreRunE`, if `!fs.Changed(name)` and an env var is set, `fs.Set(name, value)`. Wire: `TASKCTL_DEBUG`→debug, `TASKCTL_CONFIG_FILE`→config, `TASKCTL_OUTPUT_FORMAT`→output, `TASKCTL_NO_INPUT`→no-input.
- **`PersistentPreRunE`** ports the urfave `Before` (`cmd.go:134-179`) verbatim in behavior: config load (same `ErrConfigNotFound` tolerance), `--set` var application, slog level, quiet mode, then output resolution:
  1. `--output` explicitly set (`Flags().Changed("output")`) → use it;
  2. `--raw` → `FormatRaw`;
  3. config `output:` value;
  4. **else `FormatDefault`** (the new default — was `prefixed`).
  - Accept `"cockpit"` as a **deprecated alias** for `default` (config files in the wild say `output: cockpit`): map it to `FormatDefault` with a one-line `slog.Warn`.
  - Downgrade `FormatDefault` → `FormatPrefixed` when `!tui.Interactive(os.Stdout)` (unchanged rule, user-confirmed).
  - `nonInteractive` (`cmd.go:45-47`) becomes `cmdutil.NonInteractive(st, cmd) bool`.

Internal type/file names in `internal/output/cockpit.go` stay as-is (it *is* the dashboard implementation; only the user-facing format string changes).

### A3. Port every subcommand (into its own package per A0, `NewCommand(st *cmdutil.State) *cobra.Command`)

Signature change everywhere: `Action(c *cli.Context)` → `RunE(cmd *cobra.Command, args []string) error`; `c.Args().Slice()`→`args`; `c.Bool("x")`→`cmd.Flags().GetBool("x")` (persistent flags resolve through `Flags()` on any descendant); `c.IsSet`→`Flags().Changed`; per-command `Before`→`PreRunE`; `c.Context`→`cmd.Context()`.

- **`run.go`**: `run` command, `Args: cobra.MinimumNArgs(1)`. **Delete the duplicate local `--dry-run`/`--summary`** (persistent covers them; this was a latent value-misroute). Keep the `task` subcommand. `--` split: replace the `targetNames`/`taskArgs` literal-`--` scanning with a single helper `func splitArgsAtDash(cmd *cobra.Command, args []string) (targets, passArgs []string)` using `cmd.ArgsLenAtDash()`; keep the `pipeline` keyword stripping of `targetNames`. `buildTaskRunner(ctx, ...)` takes `cmd.Context()`; the runner-cancel goroutine stays.
- **`cmd.go` root `rootRunE`**: same branch as `rootAction` (`cmd.go:218-263`) — args present → `runTargets`; else non-interactive/no-TTY error; else `tui.Select` over `buildSuggestions`. Bare `taskctl <task>` keeps working: cobra falls through to root `RunE` when the first arg matches no subcommand (same shadowing caveat as today for tasks named like subcommands).
- **`list.go`**: port `list` + `tasks`/`pipelines`/`watchers` subcommands. No local flags needed — persistent `--output` now works in any position (**fixes #5**: `taskctl list --output json`).
- **`show.go`**, **`graph.go`** (keep alias `g`, local `--lr`; `cli.ShowCommandHelp`→`cmd.Help()`), **`init.go`** (local `--dir`), **`watch.go`** (`Args: cobra.MinimumNArgs(1)` replaces its `Before`), **`validate.go`** (`Args: cobra.ExactArgs(1)`), **`skill.go`** (`install` with `--global`/`--force`; `SetSkillTemplate` untouched).
- **Delete `cmd/completion.go`** — cobra ships a native `completion bash|zsh|fish|powershell` subcommand.

### A4. Completions (native + dynamic)

- Root + `run` + `show` + `watch` + `graph`: `ValidArgsFunction` returning task/pipeline names — load config with the `--config` flag value (mirror of today's `BashComplete` at `cmd.go:69-79`, reusing `buildSuggestions`), escape `:` as before, `cobra.ShellCompDirectiveNoFileComp`.
- `root.RegisterFlagCompletionFunc("output", ...)` → `default`, `prefixed`, `raw`, `json`.
- `tasks.yaml` `update-completers` task: regenerate via `go run . completion bash > autocomplete/bash_completion.bash` and same for zsh (identical command shape to today — cobra prints to stdout); add fish if desired. Commit regenerated artifacts.

### A5. Did-you-mean

- Cobra's built-in subcommand suggestions stay off for the root (root has `RunE`, so unknown args fall through to it) — instead put closest-match into the real error path: in `runTarget` (`run.go:130-142`), extend `unknown task or pipeline %q` with `(did you mean %q?)` using a new helper `closestTarget(name string, cfg *config.Config) string` — small Levenshtein (~20 lines, threshold distance ≤ len/2 or ≤3) over `maps.Keys(cfg.Tasks)` + `maps.Keys(cfg.Pipelines)`. Lives in `cmd/run.go` or a tiny `cmd/suggest.go`.

### A6. Grouped, styled help

- `cobra.Group`s on root: `Run` (run, watch), `Inspect` (list, show, graph, validate), `Setup` (init, completion, skill). Set `GroupID` per command; `root.SetHelpCommandGroupID`/`SetCompletionCommandGroupID` to `Setup`.
- Modest template styling via `root.SetUsageTemplate`: bold section headers using `internal/tui` styles; add an `Examples` field on root (`taskctl prepare`, `taskctl list --output json`, `taskctl run test -- -v`). Don't over-engineer — cobra's default template with groups already reads well.
